NAME

up_ts_heya_tiryo_his_start_ymd2.sql

DESCRIPTION

賃料履歴(TS_HEYA_TINRYO_HIS)の情報を更新する

今回、END_FLG=1（前回データ提供があったが、今回ない部屋） の履歴情報の情報開始日START_YMDを求める

再スタートしていない物件(END_FLG=1が過去ない物件）は、 最小のデータ供給日がSTART_YMDとなる。(END_FLGでもSTART_FLGでもない） (START_FLG!=1,END_FLG!=1のときのTS_HEYA_TINRYO_HIS.MIN(TIN_HIS_YMD)) が物件の空室スタートの日となる
