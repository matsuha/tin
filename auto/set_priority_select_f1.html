NAME

set_priority_select_f1.sql

DESCRIPTION

賃料マスタ(TS_HEYA_TINRYO)の優先度参考フラグを設定する 最優先フラグ（PRIORITY)はいきなり更新せず、参考FLG （PRIORITY_SELECT)に設定する）

ルール1

同一建物、同一部屋、複数問合せＮＯの場合、最小の賃料を選択 暫定フラグ　 PRIORITY_SELECT = '3'　を設定

ルール2

上記処理のPRIORITY_SELECT = '3'を対象として TS_TATEMONO.KOSU 総戸数がＮＵＬＬでない物件を１つ選択、（仮にMAX取得としている） PRIORITY_SELECT = '2'を設定

ルール3

上記処理のPRIORITY_SELECT = '2'を対象として 設備保有項目(TS_HEYA_SETUBI.SETUBI_CNT)が一番多い物件を指定 PRIORITY_SELECT = '1'を設定
