NAME

ins_ts_tatemono_dtl.sql

DESCRIPTION

建物詳細マスタ(TS_TATEMONO_DTL)の情報をインサートする

PRIMARY KEY(TATEMONO_CD,TOIAWASE_NO,KYOKYU_CD)

ワークテーブルに格納してある建物データを入力する

ts_tatemono_dtlには、IDO,KEIDOの座標カラムがあるので後からセットする IDO,KEIDOは１０進数ミリ秒単位のデータを保有している。

ゼンリンAPIの基本リターンデータの桁と同じである。 国土地理院の住所コード１〜４までのデータを 利用して取得し設定してある。

ゼンリン地図APIではJUSHOCD1-4 を順番にTOD SHK OAZ AZCで指定する（例）
addr5.cgi?pos=1&cnt=1&enc=SJIS&tod=14&shk=152&oaz=031&azc=003

また上記IDO,KEIDOカラムを利用してAABSX,AABSYの２つの エリア２５カラムにデータを変換する

コンバート用に関数が２つ用意してあるので下記のように実行する。

update ts_tatemono_dtl set aabsx = keido2aabsx(keido)
update ts_tatemono_dtl set aabsy = ido2aabsy(ido)
